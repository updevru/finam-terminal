# Spec: Обновление Go до 1.26 и внедрение нововведений

## Problem
Проект использует Go 1.24.0 (toolchain go1.24.3). Go 1.26 вышел в феврале 2026 и содержит значительные улучшения производительности, новые языковые конструкции и модернизированные API стандартной библиотеки. Необходимо обновить версию и внедрить все стабильные нововведения.

## Solution
Обновить Go до версии 1.26, применить автоматическую модернизацию кода через `go fix`, вручную внедрить новые API и языковые конструкции там, где `go fix` не справляется, обновить зависимости.

## Requirements

### Обновление версии
- Обновить `go.mod`: директиву `go` до `1.26.0`, удалить `toolchain`
- Обновить все зависимости (`go get -u ./...`, `go mod tidy`)
- Убедиться, что проект компилируется и все тесты проходят

### Автоматическая модернизация (`go fix ./...`)
- `rangeint` — заменить 3-clause `for` циклы на `range` over int
- `minmax` — заменить `if/else` паттерны на `min()`/`max()`
- `stringscut` — заменить `strings.Index` на `strings.Cut` где применимо
- `any` — заменить `interface{}` на `any` (если найдутся)
- `forvar` — убрать избыточное переобъявление переменных цикла
- `fmtappendf` — заменить `[]byte(fmt.Sprintf())` на `fmt.Appendf`
- `newexpr` — заменить helper-функции `newT(x)` на `new(x)`

### Ручное внедрение новых API
- `errors.AsType[T]()` — типобезопасная замена `errors.As` где используется
- `reflect.Type.Fields()` / `Methods()` — итераторы вместо индексных циклов в тестах
- `new(expr)` — использовать для создания указателей на значения (протобуф, JSON)
- `bytes.Buffer.Peek()` — использовать если есть подходящие паттерны
- Самореферентные generic типы — применить если есть подходящие интерфейсы

### Не включаем (экспериментальные)
- `simd/archsimd` (GOEXPERIMENT=simd)
- `runtime/secret` (GOEXPERIMENT=runtimesecret)
- Goroutine leak profile (GOEXPERIMENT=goroutineleakprofile)

## Acceptance Criteria
- [ ] `go.mod` указывает `go 1.26.0`
- [ ] `go build ./...` компилируется без ошибок
- [ ] `go vet ./...` без предупреждений
- [ ] `go test ./...` — все тесты проходят
- [ ] `go fix -diff ./...` не показывает изменений (все модернизации применены)
- [ ] Новые API Go 1.26 используются где применимо
- [ ] Все зависимости обновлены

## Edge Cases
- Зависимости могут не поддерживать Go 1.26 — проверить совместимость
- `go fix` может создать конфликты при перекрывающихся изменениях — запускать повторно
- `image/jpeg` — новая реализация encoder/decoder может дать другой бинарный выход (не затрагивает наш проект)
- `net/url.Parse()` теперь отклоняет URL с двоеточиями в host — проверить, нет ли таких в коде

## Dependencies
- Go 1.26 установлен в системе
- Все прямые зависимости совместимы с Go 1.26
