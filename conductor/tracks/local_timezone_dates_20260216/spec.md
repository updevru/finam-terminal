# Spec: Отображение дат в таймзоне пользователя

## Problem

Все даты, получаемые из Finam Trade API через protobuf (`timestamppb.AsTime()`), возвращаются в UTC. При отображении в UI они выводятся как есть — без конвертации в локальную таймзону пользователя. В результате на компьютере с московским временем (UTC+3) даты отображаются на 3 часа раньше реального времени.

Затронутые места:
- Таблица истории сделок (`render.go:186`) — `t.Timestamp.Format("01-02 15:04")`
- Таблица активных ордеров (`render.go:268`) — `o.CreationTime.Format("01-02 15:04")`

При этом часы в статусбаре (`render.go:333`) показывают `time.Now()`, что уже в локальном времени — несоответствие.

## Solution

Конвертировать все временные метки из UTC в локальную таймзону системы при получении из API (в `api/client.go`), используя `.AsTime().Local()` вместо `.AsTime()`. Это обеспечит единообразное отображение всех дат в таймзоне пользователя.

## Requirements

### Functional
- Все даты в таблицах History и Orders должны отображаться в локальной таймзоне пользователя
- Часы в статусбаре и даты в таблицах должны быть в одной таймзоне
- Формат отображения дат не меняется (`"01-02 15:04"`)

### Technical
- Использовать `time.Time.Local()` или `.In(time.Local)` для конвертации
- Конвертацию выполнять на уровне API клиента при маппинге protobuf -> models
- Не вводить дополнительных зависимостей

## Acceptance Criteria

- [ ] Даты сделок в таблице History отображаются в локальной таймзоне
- [ ] Даты ордеров в таблице Orders отображаются в локальной таймзоне
- [ ] Дата открытия счёта конвертируется в локальную таймзону
- [ ] Временные метки котировок конвертируются в локальную таймзону
- [ ] Часы в статусбаре и даты в таблицах согласованы по таймзоне

## Edge Cases

- Системы с нестандартной таймзоной (UTC+5:30, UTC-9:30) — стандартная Go `time.Local` обрабатывает корректно
- Сервер отдаёт не-UTC время — маловероятно, protobuf timestamps всегда UTC

## Dependencies

- Нет дополнительных зависимостей, используется стандартная библиотека Go `time`
