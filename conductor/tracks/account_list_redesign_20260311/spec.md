# Spec: Редизайн списка счетов — двухстрочный формат

## Problem

Текущая таблица счетов слева содержит три столбца (ID, Type, Equity). Столбец "Type" занимает место, но не несёт критической информации при ежедневной работе. Большие суммы оценки портфеля могут не помещаться в 30-символьную панель. Также отсутствует отображение PnL (изменения оценки), что заставляет переключаться на другие вкладки.

## Solution

Переделать отображение каждого счёта на двухстрочный формат:
- **Строка 1:** Номер счёта (ID)
- **Строка 2:** Оценка портфеля (Equity) + Unrealized PnL с цветовой индикацией

Столбец "Type" убирается. Изменение отображается как абсолютное значение. Зелёный цвет для положительного PnL, красный — для отрицательного.

### Визуальный макет

```
 12345678
   1 234 567.89  +15 000.00
 87654321
     543 210.00  -2 100.50
```

## Requirements

### Functional
- Убрать столбец "Type" из таблицы счетов
- Каждый счёт занимает 2 строки: ID на первой, Equity + PnL на второй
- Equity форматируется с разделителями тысяч (пробелы) и 2 десятичными знаками
- UnrealizedPnL отображается как абсолютное значение со знаком (+/-)
- Цвет PnL: зелёный (положительный), красный (отрицательный), серый (нулевой)
- Выделение (selection) должно захватывать обе строки выбранного счёта
- Навигация Up/Down переключает по счетам (не по строкам)

### Technical
- Ширина панели остаётся 30 символов
- Используется существующее поле `UnrealizedPnL` из `models.AccountInfo`
- Форматирование чисел с пробелами-разделителями тысяч (русский формат)

## Acceptance Criteria

- [ ] Столбец "Type" не отображается
- [ ] Каждый счёт занимает ровно 2 строки в таблице
- [ ] Equity отображается с разделителями тысяч
- [ ] UnrealizedPnL отображается с правильным знаком и цветом
- [ ] Up/Down переключают между счетами (перескакивая по 2 строки)
- [ ] Ошибочные счета (LoadError) корректно отображаются в новом формате
- [ ] Выделение подсвечивает обе строки выбранного счёта

## Edge Cases

- Счёт с `LoadError` — показать ID + "[error]" на второй строке
- Нулевой PnL — серый цвет, отображать "0.00"
- Пустой/невалидный Equity — показать "—"
- Один счёт — навигация не ломается
- Очень длинный ID — обрезка или уменьшение отступов

## Dependencies

- Существующее поле `models.AccountInfo.UnrealizedPnL`
- `tview.Table` API для мульти-строчного рендеринга
